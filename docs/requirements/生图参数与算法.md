# 生图参数与算法

当前项目里的生图算法可以理解为一条流水线：

`上传图片 → 裁切 → 缩放到豆图尺寸 → 选择/限制调色板 → 每个像素匹配最近拼豆色 → 可选平滑/抖动 → 统计用料`

核心代码在这些文件里：

- `lib/services/pattern_generation_service.dart`
- `lib/algorithms/color_reducer.dart`
- `lib/services/palette_constraint_service.dart`
- `lib/services/image_service.dart`

## 一、现在已经暴露给用户的参数

### 裁切比例

当前有：

- `9:16`
- `3:4`
- `1:1`
- `4:3`
- `16:9`
- `Free`

影响：决定图片进入算法前保留哪些区域。  
对人像来说，裁切比颜色算法还重要。人物脸太小，后面再怎么调色也救不回来。小尺寸拼豆图建议裁成近景，让主体占画面 60% 以上。

代码位置：`lib/models/product_template.dart`

### 图纸尺寸 / 豆子数量

当前模板大概是：

- 小鼻嘎：`16 x 16`
- 冰箱贴：`24 x 24`
- 钥匙扣：`32 x 32`
- 大钥匙扣：`40 x 40`
- 杯垫：`48 x 48`
- 装饰画：`64 x 64`
- 行李牌：`64 x 96`
- 自定义：`8-128`

影响：决定最终像素数，也就是豆子数量。

尺寸越小，越像图标；尺寸越大，越像照片。  
人像类大致可以这样看：

- `16x16`：只能表达轮廓和大色块
- `24x24`：适合简单头像，但五官会弱
- `32x32`：勉强能做人像
- `48x48`：开始比较像样
- `64x64+`：细节明显改善，但用豆量上升

代码位置：`lib/models/product_template.dart`

### 颜色数量限制

当前有：

- 不限制
- `8`
- `16`
- `24`
- `32`

影响：决定最终图纸最多使用多少种拼豆色。

这是目前对观感影响最大的参数之一。

- `8色`：很干净，但容易海报化，肤色和阴影会丢
- `16色`：适合 MVP 默认值，复杂度可控
- `24色`：人像更推荐，肤色、头发、背景层次更好
- `32色`：更接近原图，但用料更复杂
- `不限制`：颜色最丰富，但可能噪点多、用料清单很散

当前逻辑已改成了“图片感知选色”：不是简单取品牌色板前 N 个，而是先看图片里哪些颜色最常被匹配，再选最有用的 N 个颜色。

代码位置：`lib/services/palette_constraint_service.dart`

### 颜色品牌 / 调色板

影响：决定算法可以使用哪些真实拼豆颜色。

如果品牌色板里肤色少，人像会偏黄、偏红、偏灰；如果缺少深浅过渡色，头发、阴影会断层。  
所以生图质量不只取决于算法，也取决于调色板质量。

代码位置：`lib/services/pattern_generation_service.dart`

### 颜色平滑

当前 UI 里叫“颜色平滑”，代码里对应 `smoothingEnabled`，实际作用是开启/关闭误差扩散 dithering。

影响：

- 开启：渐变更自然，照片感更强，适合人像、风景
- 关闭：色块更干净，边缘更硬，适合 logo、图标、卡通图

人像推荐先试：`24色 + 平滑开`。  
如果觉得脸上颗粒太多，再试：`16色 + 平滑关` 或 `24色 + 平滑关`。

代码位置：`lib/algorithms/color_reducer.dart`

## 二、代码里已有但新版 UI 暂时没有完全暴露的参数

### 匹配算法 MatchingAlgorithm

旧模型里有三个：

- `euclidean`
- `cie94`
- `cie2000`

新版生成流程现在固定使用 `cie2000`。

影响：决定“一个图片像素应该匹配哪颗拼豆色”。

- `Euclidean`：直接算 RGB 距离，快，但不够符合人眼感知
- `CIE94`：更接近人眼
- `CIE2000`：当前默认，通常最适合照片和人像

这个参数一般不建议给普通用户看，可以作为高级调试项。

代码位置：`lib/models/project.dart`

### 抖动强度 ditheringHardness

新版生成里固定是 `50`。

影响：控制误差扩散的强度。

- `0`：基本不扩散，色块更平
- `20-35`：比较干净，适合脸部
- `50`：当前默认，细节和平滑折中
- `70-100`：渐变更强，但容易有噪点、杂色豆

这个很值得做成高级参数，或者做成三个预设：

- 干净：`20`
- 均衡：`50`
- 细节：`75`

代码位置：`lib/services/pattern_generation_service.dart`

### 缩放插值 interpolation

当前固定是 `img.Interpolation.linear`。

影响：原图缩小成拼豆尺寸时，怎么压缩细节。

- `nearest`：边缘硬，容易锯齿
- `linear`：当前默认，比较平滑
- `cubic`：可能更柔和，但也可能糊
- `average`：缩小照片时可能更稳定，值得测试

这个参数对小尺寸图纸影响很大。比如把一张高清人像压成 `32x32`，插值方式会直接决定五官是“糊成一团”还是“保留大致结构”。

代码位置：`lib/services/image_service.dart`

### fitImage / centerImage

代码里缩放图片时支持：

- `fit = true`
- `center = true`

新版流程基本固定为保持比例并居中。

影响：

- `fit=true`：完整放入画布，不变形，但可能留透明边
- `fit=false`：可能按原尺寸/裁切逻辑处理，当前不适合 MVP
- `center=true`：主体居中
- `center=false`：从左上角开始贴图，不适合普通用户

代码位置：`lib/services/image_service.dart`

## 三、当前算法里还没有，但很值得新增的调优参数

### 亮度 / 对比度预处理

影响：让图像在变成低分辨率豆图前更清晰。

人像经常需要轻微增强对比度，否则缩小后五官会没掉。  
建议新增预设：

- 原图
- 清晰
- 柔和
- 高对比

### 锐化 sharpen

影响：缩小前增强边缘。

对眼睛、嘴巴、发丝有帮助，但过强会产生脏点。  
建议只做轻度锐化，不要给用户一个 0-100 的复杂滑杆。

### 背景处理

影响：背景是否参与颜色选择和豆子统计。

如果背景很复杂，算法会把颜色额度浪费在背景上，人物反而变差。  
后续可以做：

- 保留背景
- 弱化背景
- 去背景
- 纯色背景

### 肤色保护 / 主体优先

影响：人像质量。

现在颜色限制是按全图像素统计。如果背景面积大，背景颜色会抢占名额。  
更好的逻辑是：脸部/主体区域权重更高，背景权重更低。

### 边缘增强

影响：拼豆图的可读性。

可以在生成后对人物轮廓、眼睛、嘴巴附近做轻微深色边缘强化。  
这会让小尺寸图纸更“像”，但如果过度会变成漫画风。

## 四、我建议第一版重点研究的参数组合

先不要一下子开放太多参数。建议 MVP 里做三个“效果模式”：

### 清晰模式

- 颜色：`16`
- 平滑：关或低强度
- 适合：logo、卡通、简单头像

### 均衡模式

- 颜色：`24`
- 平滑：开
- 抖动强度：`50`
- 适合：默认、人像、照片

### 细节模式

- 颜色：`32`
- 平滑：开
- 抖动强度：`60-75`
- 适合：大尺寸装饰画

建议下一步做一个“算法对比页”：同一张图片一次生成 4 个预览，比如 `16色平滑关 / 16色平滑开 / 24色平滑开 / 32色平滑开`。这样会非常直观看到每个参数的效果，也方便后面决定 UI 里到底给用户暴露哪些参数。

## 六、后续建议

### 1. 先做“算法对比页”，不要先堆参数

在产品和 UI 还没完全定型前，最有价值的是快速看效果。建议新增一个内部调试页或隐藏入口：

- 同一张图一次生成多个版本
- 每个版本显示参数组合
- 支持点击某个版本进入结果页

首批对比组合建议：

- `16色 + 平滑关`
- `16色 + 平滑开`
- `24色 + 平滑开`
- `32色 + 平滑开`

这样可以快速判断用户默认应该看到哪种效果。

### 2. 第一版用户侧不要暴露太多算法词

用户不一定理解 `CIE2000`、`dithering`、`interpolation`。建议 UI 上只给：

- 图纸大小
- 颜色数量
- 效果模式
- 颜色品牌

把复杂参数藏在“效果模式”后面。

### 3. 把“颜色平滑”升级成“效果模式”

当前“颜色平滑”是一个单独开关，但真实效果取决于颜色数、抖动强度、缩放方式、预处理。建议后续把它升级为：

- 干净
- 均衡
- 细节

每个模式内部绑定不同参数组合。

### 4. 人像优化应该成为单独方向

拼豆 app 很可能会大量处理头像、情侣照、宠物照、二次元头像。对人像来说，单纯颜色匹配不够，后续建议重点研究：

- 主体检测
- 背景弱化
- 肤色保护
- 轻锐化
- 眼睛和嘴巴区域增强

### 5. 调色板质量需要产品化管理

如果真实品牌色板不完整，算法会被限制住。建议后续给每个品牌色板增加质量检查：

- 是否有足够肤色
- 是否有足够灰阶
- 是否有深浅红/棕/黑发色
- 是否有常用阴影色
- 每个颜色是否有真实库存或购买链接

### 6. 建议形成默认参数

当前建议的第一版默认值：

| 场景 | 尺寸 | 颜色数 | 平滑 | 抖动强度 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 普通人像 | `48x48` 或以上 | `24` | 开 | `50` | 第一版推荐默认 |
| 小头像 / 钥匙扣 | `32x32` | `16` 或 `24` | 开 | `35-50` | 避免噪点过多 |
| Logo / 图标 | `32x32` 或以上 | `8` 或 `16` | 关 | `0` | 保持边缘干净 |
| 装饰画 | `64x64` 或以上 | `32` | 开 | `60-75` | 允许更多细节 |
| 复杂照片 | `64x64` 或以上 | `32` 或不限制 | 开 | `50` | 需要接受用料复杂 |

### 7. 下一步研发顺序建议

建议按这个顺序推进算法研究：

1. 做算法对比页，快速观察参数组合效果。
2. 暴露内部调试参数：颜色数、平滑、抖动强度、缩放插值。
3. 增加预处理：亮度、对比度、轻锐化。
4. 做人像专项：背景弱化、肤色保护、主体优先。
5. 固化 3 个用户可理解的效果模式。
6. 再回到 UI，把复杂参数收敛成简单选择。
