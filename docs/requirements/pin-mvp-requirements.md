# Pin MVP 需求文档

日期：2026-06-10
状态：草稿
产品阶段：MVP

## 概要

Pin 是一个移动端客户端，用于把用户选择的一张照片转换成拼豆图纸，并生成可用的用料清单。第一版需要完成核心制作流程：上传照片、裁切照片、选择商品化尺寸模板、选择颜色方案、生成图纸、轻量编辑结果并保存。

MVP 应该像一个简单的手作工具，而不是专业图片编辑器、市场平台或社交应用。

## 目标

- 让用户无需理解技术化的拼豆尺寸，也能从一张照片生成拼豆图纸。
- 让用户选择熟悉的商品/用途模板，例如钥匙扣、杯垫、行李牌或装饰画。
- 生成清晰的图纸预览和用料清单，包含颜色编号和所需拼豆数量。
- 允许用户在生成后进行轻量修正：取色、画笔、橡皮、撤销、恢复，并重新计算用量。
- 将完成后的项目保存到 app 本地。

## 非目标

- 账号登录或云同步。
- 社区、模板市场或内容分享流。
- 购物车、下单或拼豆购买流程。
- 多图片批量生成。
- PDF 分板打印。
- 高级图片滤镜、AI 修图、亮度/对比度调整或背景移除。
- 带图层、魔棒、套索、填充桶、选区或区域变形的完整专业画布编辑器。
- 在同一张生成图纸中混用多个品牌色板。

## 已确认决策

- 尺寸选择是商品/用途模板，不向用户展示原始技术参数。
- 第一版包含轻量编辑。
- 第一版每个项目只允许上传一张图片。
- 第一版每次生成只使用一个颜色品牌。
- 保存表示保存到 app 内。导出/分享可以作为二级操作补充。

## 主流程

```text
启动 / 首页
  -> 点击“上传照片”
  -> 打开系统相册选择器
  -> 用户选择一张图片
  -> 裁切照片
  -> 选择商品尺寸模板
  -> 选择颜色设置
  -> 生成图纸
  -> 查看图纸 + 用料清单
  -> 可选：轻量编辑
  -> 保存项目
```

## 页面需求

### 1. 上传照片

目的：从单张图片开始创建新的拼豆项目。

需求：
- 点击“上传照片”后立即打开系统相册/图片选择器。
- 只能选择一张图片。
- 选择成功后，直接进入裁切页面。
- 如果用户取消选择，停留在当前页面，不创建项目。
- 如果相册权限被拒绝，展示可恢复的权限提示。
- 如果图片解码失败，展示清晰错误，并允许用户重新选择图片。

状态：
- 空闲。
- 正在打开选择器。
- 图片已选择。
- 权限被拒绝。
- 图片加载失败。

### 2. 图片裁切

目的：在拼豆转换前整理照片构图。

需求：
- 在裁切 UI 中展示已选择照片。
- 支持固定比例：9:16、3:4、1:1、4:3、16:9。
- 支持通过拖拽裁切框进行自由裁切。
- 默认比例：优先使用上一次选择的比例；如果没有历史记录，则默认 1:1。
- 确认裁切后生成裁切图片，并进入尺寸选择页面。
- 返回时应回到上传/首页，并保留已选择的原图，除非用户退出当前项目。

不在范围内：
- 滤镜。
- 旋转。
- 亮度/对比度。
- AI 图片增强。

状态：
- 裁切就绪。
- 已选择比例。
- 自由裁切。
- 正在确认裁切。
- 裁切失败。

### 3. 选择参数 - 尺寸

目的：让用户选择想制作的实体产品，同时由 app 在内部映射成拼豆尺寸。

需求：
- 展示裁切后图片预览。
- 以可选卡片形式展示商品/用途模板。
- 每个模板包含展示名称、简短副标题、物理尺寸、内部拼豆宽度、内部拼豆高度、默认裁切比例和预计拼豆数量。
- 选择模板后，更新预计拼豆数量和内部输出尺寸。
- 为高级用户提供自定义选项。
- 自定义选项允许输入拼豆宽度和高度。
- 自定义尺寸必须限制最小和最大拼豆尺寸。
- CTA 文案：“选择颜色”。

初始模板集合：

| 模板 | 展示示例 | 内部含义 |
| --- | --- | --- |
| 小挂件 | 小挂件 / 2.6cm | 小型配饰图纸 |
| 冰箱贴 | 冰箱贴 / 4.0cm | 小型方形/竖向冰箱贴 |
| 钥匙扣 | 钥匙扣 / 5.3cm | 常见便携挂件 |
| 大钥匙扣 | 大钥匙扣 / 7.0cm | 更大的配饰图纸 |
| 杯垫 | 杯垫 / 8.2cm | 方形实用作品 |
| 装饰画 | 装饰画 / 11.0cm | 小型展示画 |
| 行李牌 | 行李牌 / 14.0cm | 较大的矩形产品 |
| 自定义 | 自定义 / XX cm | 用户自定义拼豆尺寸 |

开放决策：
- 每个模板的准确拼豆宽高和物理尺寸需要产品/业务侧确认。

状态：
- 已选择模板。
- 已选择自定义。
- 自定义尺寸无效。
- 可以继续。

### 4. 选择参数 - 颜色

目的：在生成前选择颜色约束。

需求：
- 展示裁切后图片预览。
- 选择一个拼豆颜色品牌。
- 选择颜色数量限制。
- 颜色数量选项应为离散值：
  - 不限制
  - 8 色
  - 16 色
  - 24 色
  - 32 色
- 提供颜色平滑开关。
- 默认颜色品牌应优先使用上次使用的品牌；如果没有历史记录，则使用 app 默认品牌。
- MVP 默认颜色数量为 16 色，除非测试证明 8 色更适合手作识别。
- 默认开启颜色平滑。
- CTA 文案：“生成图纸”。
- 生成中禁止重复点击。

不在范围内：
- 多品牌混色。
- 生成前手动启用/禁用单个颜色。
- 基于库存的颜色选择。

状态：
- 已选择品牌。
- 已选择颜色限制。
- 颜色平滑开/关。
- 已请求生成。
- 色板状态无效。

### 5. 图纸生成

目的：将裁切后的图片转换成拼豆图纸。

需求：
- 展示生成/加载状态。
- 在支持的情况下，将图片处理放到 UI 线程之外执行。
- 使用已选择的模板尺寸、色板品牌、颜色数量限制和颜色平滑设置。
- 成功后进入结果页。
- 失败后展示重试选项。
- 如果用户在生成过程中离开，应取消任务，或在结果返回时安全忽略该结果。

状态：
- 正在生成。
- 成功。
- 失败并可重试。
- 已取消/已离开。

### 6. 结果页

目的：展示生成后的拼豆图纸和用料清单。

需求：
- 展示拼豆图纸预览。
- 展示拼豆总数。
- 展示用料清单，包含：
  - 颜色色块
  - 颜色编号
  - 颜色名称
  - 所需数量
- 默认按数量从高到低排序用料清单。
- 点击颜色后进入替换颜色操作。
- 长按颜色后提供删除/移除该颜色的操作。
- 替换或删除颜色后，重新计算图纸像素和用量清单。
- 底部操作：
  - 编辑
  - 保存

开放决策：
- 长按删除是否需要确认。建议：当删除颜色影响超过 5% 的拼豆时进行确认。

状态：
- 结果就绪。
- 替换颜色流程。
- 删除颜色确认。
- 正在重新计算用量。
- 正在保存。
- 保存成功。
- 保存失败。

### 7. 轻量编辑器

目的：让用户修正明显的生成问题，同时避免把 MVP 变成完整绘图应用。

需求：
- 从结果页进入。
- 展示可编辑的拼豆图纸画布。
- 工具：
  - 取色器
  - 当前颜色
  - 画笔
  - 橡皮
  - 笔刷大小
  - 撤销
  - 恢复
  - OK/确认
- 画笔支持小范围绘制。
- 橡皮根据项目规则清除拼豆或将拼豆变为透明。
- 取色器选择被点击格子下的拼豆颜色。
- 撤销/恢复适用于编辑操作。
- 确认编辑后返回结果页，并更新用料。
- 取消编辑后返回结果页，不应用改动。

不在范围内：
- 图层。
- 套索。
- 魔棒。
- 填充桶。
- 区域选择。
- 编辑器内批量替换。

状态：
- 编辑中。
- 画笔激活。
- 橡皮激活。
- 取色器激活。
- 撤销可用/不可用。
- 恢复可用/不可用。
- 确认编辑。
- 放弃编辑。

## 推荐实施阶段

### M1：生成闭环

交付完整的非编辑项目流程：
- 上传照片
- 裁切照片
- 选择商品模板
- 选择颜色设置
- 生成图纸
- 查看结果
- 查看用料清单
- 保存本地项目

### M2：轻量编辑

在 M1 稳定后加入编辑器：
- 从结果页打开编辑器
- 取色器/当前颜色
- 画笔
- 橡皮
- 笔刷大小
- 撤销/恢复
- 确认/取消编辑
- 编辑后重新计算用量

## 测试计划

### 单元测试

- 商品模板能正确映射到拼豆宽度、高度、物理尺寸和预计数量。
- 自定义尺寸会拒绝无效的最小/最大值。
- 颜色数量选项能映射到预期的生成约束。
- 图纸生成会返回所选模板对应的宽高。
- 用量统计能正确统计颜色数量。
- 替换颜色会更新像素和用量。
- 删除颜色会更新像素和用量。
- 编辑器撤销/恢复能还原之前的像素状态。

### Widget 测试

- 取消上传后，app 保持在起始状态。
- 成功选择图片后进入裁切页。
- 确认裁切后进入尺寸选择页。
- 选择模板后启用“选择颜色”按钮。
- 无效自定义尺寸会禁用下一步 CTA。
- 颜色页会阻止重复点击生成。
- 生成失败时展示重试。
- 结果页展示拼豆总数和颜色用量行。
- 保存失败是可见且可恢复的。

### 集成 / 手动 QA

```text
主路径：
上传照片 -> 裁切 1:1 -> 选择钥匙扣 -> 选择 16 色 -> 生成 -> 保存

自定义尺寸路径：
上传照片 -> 自由裁切 -> 自定义尺寸 -> 生成 -> 验证输出尺寸

编辑器路径：
生成图纸 -> 打开编辑器 -> 取色 -> 绘制 -> 撤销 -> 恢复 -> OK -> 用量更新

失败路径：
拒绝相册权限 -> app 展示恢复提示
色板不可用 -> app 使用兜底方案或展示可恢复错误
生成失败 -> 重试可用
```

## 失败模式

| 失败 | 预期行为 |
| --- | --- |
| 用户拒绝相册权限 | 展示权限恢复提示 |
| 用户取消图片选择 | 停留在上传/首页状态 |
| 图片解码失败 | 展示“图片无法加载”并允许用户重试 |
| 裁切输出为空 | 阻止继续，并展示裁切错误 |
| 自定义尺寸过大 | 阻止继续，并说明尺寸限制 |
| 色板加载失败 | 使用缓存/本地色板，或展示可恢复错误 |
| 生成耗时过长 | 保持加载状态，并阻止重复生成 |
| 生成失败 | 展示重试 |
| 用户在生成中离开 | 取消任务或忽略过期结果 |
| 保存失败 | 保持结果可见，并展示重试 |
| 编辑导致图纸为空 | 阻止确认，或询问用户是否放弃 |

## 验收标准

- 用户可以使用一张图片完成从上传到保存项目的完整流程。
- 生成图纸的尺寸与所选商品模板匹配。
- 用料清单展示准确的拼豆总数和每种颜色数量。
- 用户可以进入轻量编辑器，修改至少一个拼豆颜色，撤销/恢复，确认后看到更新后的用量。
- app 能处理相册取消、权限拒绝、图片解码失败、生成失败和保存失败，不出现空白页或崩溃。
- 第一版范围明确排除登录、云同步、市场、购物、完整编辑器和 PDF 打印。

## 开放问题

- 每个商品模板的准确拼豆尺寸和物理尺寸是多少？
- 长按删除颜色是否应基于受影响拼豆数量进行确认？
- “保存”是否也应该导出到系统相册，还是导出/分享应作为独立操作？
- 面向中国用户时，默认色板品牌应该是什么？
- MVP 是否应该展示“不限制颜色”，还是默认选项从 8 色开始？
